# Security Policy

## Reporting a vulnerability

If you discover a security issue in ff_sdk (or in robots controlled by it),
please report it **privately** — do not open a public issue.

- Email the FF Robotics Apps team with subject `[ff_sdk security]`.
- Include: affected version, a description, and reproduction steps if possible.
- Please allow a reasonable window for a fix before any public disclosure.

We take robot-control safety seriously — vulnerabilities that could cause
unsafe motion, bypass of `e_stop`, or unauthorized robot access are treated as
highest priority.

## Supported versions

ff_sdk is pre-1.0 (`0.1.0aN`). Only the latest release receives security fixes.

## Scope

This repository publishes **documentation and examples only**. The SDK itself
ships as an encrypted, obfuscated distribution. Reports about the published
docs/examples and about the distributed SDK behavior are both welcome.
